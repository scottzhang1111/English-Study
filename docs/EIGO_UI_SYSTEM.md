# Eigo World Mobile UI System

Official mobile style reference:
LearningHubPage, GrammarTest mobile page, FlashcardPage mobile word detail, EQBottomNav.

Use these rules for Eigo World mobile screens and fixes.

## 1. Dark Fantasy Background

- Use a deep fantasy game backdrop as the page base: dark navy, violet, forest, or night-sky tones.
- Layer subtle gradients, glows, or texture to create depth without reducing readability.
- Keep the background full-page and continuous behind all content.
- Avoid plain app-gray, pure white, or flat classroom-style backgrounds.

## 2. Gold-Border Cards

- Primary content cards use dark translucent fills with warm gold borders.
- Gold borders should feel magical and premium, not harsh yellow outlines.
- Use soft shadows or inner highlights to separate cards from the background.
- Card corners should match existing Eigo Quest mobile styling.
- Keep cards compact on mobile; avoid oversized empty panels.

## 3. Mobile Page Header

- Headers should feel like an in-world quest screen, not a generic web page.
- Use a short title, optional icon, and concise supporting metadata only when useful.
- Keep header height modest so learning content appears quickly.
- Back, close, or mode actions belong in predictable top positions.
- Do not introduce large hero sections on functional learning pages.

## 4. Quest Cards

- Quest cards represent selectable journeys, stages, worlds, or learning paths.
- Use rich dark card surfaces, gold accents, clear titles, and compact progress signals.
- Show status through badges, bars, icons, or small labels rather than long explanations.
- Make the whole card tappable when it is the primary action.
- Preserve scanability: one quest card should not dominate the viewport.

## 5. Question Cards

- Question cards must prioritize the prompt, choices, feedback, and next action.
- Use the same dark card and gold-border family as the rest of Eigo World.
- Keep answer choices large enough to tap, with clear selected, correct, and incorrect states.
- Feedback should feel embedded in the card, not like a separate plain white notice.
- Avoid layout shifts when feedback appears.

## 6. Info Cards

- Info cards are for summaries, hints, stats, word details, grammar notes, or progress.
- Use restrained spacing and compact headings.
- Favor icon-plus-value, badge rows, and short explanatory text.
- Info cards should support the task, not become decorative filler.
- Never use large plain white blocks for definitions, explanations, or details.

## 7. Buttons

- Primary actions use warm gold styling or the established Eigo Quest accent treatment.
- Secondary actions stay darker, outlined, or subdued.
- Buttons must have clear tap targets and visible pressed/disabled states.
- Icon buttons should use familiar symbols and include accessible labels.
- Avoid unrelated button restyling when fixing local layout issues.

## 8. Badges

- Badges should communicate level, status, count, type, rarity, or progress.
- Use gold, gem, elemental, or muted dark treatments that fit the fantasy UI.
- Keep badge text short: examples include `New`, `Done`, `Lv 3`, `Review`, `Boss`.
- Badges should not compete with primary buttons.
- Avoid bright default browser colors.

## 9. Bottom Nav Usage

- Use EQBottomNav for main mobile app navigation.
- Keep it fixed at the bottom when moving between core Eigo World areas.
- Ensure page content has enough bottom padding so nav never covers controls.
- Do not duplicate bottom navigation inside page content.
- Hide or replace it only for focused flows where existing patterns already do so.

## 10. Do-Not Rules

- Do not add large plain white blocks to Eigo World mobile screens.
- Do not rewrite unrelated layouts while fixing a specific screen.
- Do not mix generic admin/dashboard styling into the fantasy learning UI.
- Do not introduce new visual systems unless they intentionally extend this one.
- Do not make mobile pages rely on desktop spacing, wide cards, or multi-column layouts.
- Do not cover primary actions with fixed headers, overlays, or bottom nav.
